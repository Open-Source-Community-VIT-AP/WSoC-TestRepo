Welcome to the Front End Section.

In order to make a successful pull request, you must design a web page adhering to the recommendations as given below.

It would help for you to look into the given ideas, before you proceed to work on the web page.

  • You may use BootStrap, which has been used in all of the other web pages of this project.
  • You may want to look into template inheritance and see how it has been used in this project to simplify the initial steps of the task list.
  • You should look into how to link static files from the project into the specific html in django, this will be essential to insert images into the web page.
  • Feel free to commit and push to your branch as soon as you are done with one or the other feature. This allows us to evaluate your submission better.
  • Any sort of self promotion is alright, but, it should only link to your professional works.
  • Keep a note of the fact that this web page must be created in the file named front_end_submission.html

The task list below, is in a particular order and must not be mixed up.

  • A navigation bar with the same brand, and links as the one in the home page.
  • A large image of the WSoC logo, which you can find in the static folder of the django project, to be placed as a banner for the web page.
  • A large heading for the web page which says "FRONT END"
  • Dummy text, or Lorem Ipsum text as a paragraph.
  • The following part will be in two sections. Left and Right.
    • Left : A subtitle with the text "Web Development".
    • Left : Lorem Ipsum Paragraph text under the subtitle
    • Right : An Image of the bread doggo in the static folder directory, in a square shaped format.
    • (Optional) Right : Make a caption for bread doggo.
